# エージェント（Jules）の責務と行動指針

> [!IMPORTANT]
> Julesは、タスク完了時に必ず**作業ブランチでプルリクエスト（PR）を作成し、ユーザーにレビューを依頼する**必要があります。これは、レビュー結果がNGであった場合でも、ユーザーがPR上に具体的な修正指摘（コメント）を残せるようにするためです。

このファイルは、AIエージェント（Jules）がこのリポジトリで作業する際の指針を定めたものです。

## 1. 開発プロセスの完結
Julesは、与えられたタスクに対して単にコードを書くだけでなく、以下のプロセスを完遂する責務を負います。
- 実装およびテストの実施
- セルフレビューおよび検証
- **作業ブランチでのプルリクエスト（PR）作成**
    - タスクが完了した段階で、適切なタイトルと説明文（Description）を添えて、作業内容を反映したプルリクエストを必ず作成してください。
- PR作成後の報告

## 2. プルリクエストの作成
タスクが完了した段階で、適切なタイトルと説明文（Description）を添えて、作業内容を反映したプルリクエストを必ず作成してください。
また、対応しているIssueの番号が判明している場合は、**PRの「Development」属性でPRと該当Issueを必ず紐付け**てください。

> [!IMPORTANT]
> スマホからのバイブコーディング環境での確認を容易にするため、PRの本文には必ず以下のセクションを設けてください。
> - **動作確認観点**: 何を確認したか
> - **動作確認エビデンス**: 確認結果（ログのコピペやファイルへのリンク）
>
> **記載例:**
> ### 動作確認観点
> - `npm run dev` で "Hello World" が出力されること
>
> ### 動作確認エビデンス
> ```bash
> $ npm run dev
> > pure-synth-backend@1.0.0 dev
> > spago run
> Hello World
> ```
> もしくは、エビデンスが膨大な場合は `docs/evidences/issue-NNNNN-summary/` 配下にファイルを保存し、そのリンクを貼ってください。


## 3. コミュニケーションの言語
Julesは、Julesアプリ内でのユーザーとの対話だけでなく、GitHub上でのすべてのやり取り（コミットメッセージ、プルリクエストのタイトル・説明文、コメントなど）において、**常に日本語を使用**してください。

## 4. バイブコーディング（Vibe Coding）の推進
2026年3月時点のベストプラクティスに基づき、ユーザーの「意図（バイブス）」を汲み取った自律的な提案と実装を行います。

## 5. セルフチェック項目
Julesは、PRを作成する前に以下の項目をセルフチェックしてください。
- [ ] 要求されたすべての機能が実装されているか
- [ ] 適切なテストが作成・更新され、すべてパスしているか
- [ ] 動作確認エビデンスが用意されているか
- [ ] 初心者にも配慮した適切なコメントやNOTEアノテーションが含まれているか
- [ ] PR本文に「動作確認観点」と「動作確認エビデンス」が含まれているか
